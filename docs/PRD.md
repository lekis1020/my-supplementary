> Version: 1.0.0
> 작성일: 2026-03-12
> 상태: 확정

---

## 1. 제품 개요

### 한 줄 요약
소비자가 영양제·건강기능식품의 원료별 기능성·근거·부작용·제품 비교를 신뢰할 수 있는 데이터로 확인할 수 있는 공개 웹 서비스

### 핵심 가치
- **신뢰성**: 규제기관 데이터 + 학술 근거 기반, 정식 검수(L1→L2→L3) 통과 후 게시
- **투명성**: 모든 정보에 출처·갱신일 표기, 규제 허용 표현과 학술 연구 명확 분리
- **비교 가능성**: 원료 중심 구조로 제품 간 성분·함량·근거 수준 비교

### 대상 사용자
| 사용자 | 니즈 |
|--------|------|
| **일반 소비자** | "내가 먹는 영양제가 효과 있는지, 부작용은 없는지, 더 좋은 대안은?" |
| **건강 관심층** | "원료별 근거 수준이 궁금하다. 과학적 데이터로 비교하고 싶다." |
| **복수 영양제 복용자** | "지금 먹는 것들이 중복되거나 과다하진 않은지 확인" |

## 4. 화면 구조

### 소비자용

```
/                              → 메인 (검색, 목적별 바로가기)
/ingredients                   → 원료 전체 목록 (카테고리 필터)
/ingredients/[slug]            → 원료 상세
                                  ├─ 기본 정보 (표준명, 동의어, 카테고리)
                                  ├─ 국내 허용 기능성 (식약처 인정 뱃지)
                                  ├─ 학술 연구 영역 (근거 등급 A~I)
                                  ├─ 부작용/경고/금기 (3계층)
                                  ├─ 약물 상호작용
                                  ├─ 권장 용량 (집단별)
                                  ├─ 관련 제품 리스트
                                  └─ 참고문헌
/products                      → 제품 목록 (필터·정렬)
/products/[id]                 → 제품 상세
                                  ├─ 제품 기본 정보 (제조사, 가격 등)
                                  ├─ 원료 조성 (성분표)
                                  ├─ 라벨 원문 (스냅샷)
                                  ├─ 원료별 기능성·근거 요약
                                  └─ 주의사항
/compare                       → 제품 비교 (최대 4개)
                                  ├─ 성분 함량 나란히 비교
                                  ├─ 근거 수준 비교
                                  ├─ 성분 중복 경고
                                  └─ 가격 대비 함량
/guide/interactions            → 상호작용 조회
/guide/cautions                → 특수 집단별 주의사항
/disclaimer                    → 의료 면책 조항
```

### 관리자용 (/admin)

```
/admin                         → 대시보드 (검수 큐 현황, 수집 상태)
/admin/review                  → 검수 큐 (L1/L2/L3 탭)
/admin/review/[task_id]        → 검수 상세
                                  ├─ raw 문서 보기
                                  ├─ 추출 결과 vs 이전 버전 diff
                                  ├─ 승인/반려/반송
                                  └─ 코멘트·사유 기록
/admin/collections             → 수집 작업 현황
/admin/ingredients             → 원료 관리 (CRUD)
/admin/products                → 제품 관리 (CRUD)
/admin/sources                 → 소스·커넥터 관리
```

## 5. 사용자 시나리오

### S1. 원료 조회
> 사용자가 "비타민 D"를 검색 → 원료 상세 페이지 진입 → 국내 허용 기능성("뼈의 형성과 유지에 필요") 확인 → 학술 근거 영역(면역 조절 B등급, 근력 유지 C등급) 확인 → 부작용 확인 → 관련 제품 클릭

### S2. 제품 비교
> 사용자가 비타민 D 제품 3개를 비교 도구에 추가 → 함량(1000IU vs 2000IU vs 5000IU), 가격, 추가 성분 나란히 비교 → "제품 A와 C에 칼슘이 중복됩니다" 경고 확인

### S3. 라벨 확인
> 사용자가 특정 제품 상세 페이지에서 "라벨 원문 보기" 클릭 → 라벨 스냅샷(원문 텍스트) 확인 → 성분표와 파싱된 원료 조성 대조

### S4. 부작용 조회
> 사용자가 오메가3 상세 페이지에서 부작용 탭 확인 → 1층(규제기관 경고: 수술 전 중단 권고), 2층(문헌: 위장 불편), 3층(자발보고: 신호) 구분 확인

### S5. 상호작용 확인
> 사용자가 "비타민 E + 항응고제" 상호작용 조회 → "출혈 위험 증가" 경고 + 출처(DailyMed) + 근거 수준 표시

### S6. 복수 영양제 중복 확인
> 사용자가 현재 복용 중인 영양제 3개 선택 → 합산 성분 분석 → "비타민 D 총 8000IU — 상한섭취량(4000IU) 초과" 경고

### S7. 검수자 워크플로우 (admin)
> L1 자동 검수 통과 → L2 과학 검수자에게 할당 → 논문 해석 확인 + 근거 등급 검토 → 승인 → L3 규제 검수자에게 자동 전달 → 허용 표현 확인 → 승인 → `is_published = TRUE` → 서비스 노출

### S8. 라벨 변경 감지 (admin)
> 수집기가 제품 라벨 페이지 checksum 변경 감지 → 새 raw_document 저장 → 이전 label_snapshot과 diff 표시 → L1→L2→L3 재검수 시작

## 6. 정보 표시 원칙

### 규제 표현 분리 (필수)
```
┌─────────────────────────────────────────────┐
│ 🏛️ 국내 허용 기능성 (식약처 인정)             │
│   "뼈의 형성과 유지에 필요"                    │
├─────────────────────────────────────────────┤
│ 📚 학술 문헌상 연구된 영역                     │
│   면역 조절 (근거 B)  ·  근력 유지 (근거 C)    │
├─────────────────────────────────────────────┤
│ ⚠️ 이 영역은 국내 기능성 인정 범위 밖입니다     │
└─────────────────────────────────────────────┘
```

### 부작용 3계층 표시
| 계층 | 표시 방법 | 출처 |
|------|-----------|------|
| 1층: 확정적 | 빨간색 배너 | 규제기관 라벨/공식 경고 |
| 2층: 문헌 | 주황색 카드 | RCT/체계적 문헌고찰/증례 |
| 3층: 신호 | 회색 카드 + "보고 신호" 라벨 | 자발보고 DB |

### 면책 표시 (모든 페이지)
- "이 정보는 의료 조언이 아닙니다"
- "질환 치료 목적 복용은 전문가 상담 필요"
- "제품 효과는 원료 근거와 동일하지 않을 수 있음"

## 8. 비기능 요구사항

### 성능
- 페이지 로딩: FCP < 1.5s (Vercel Edge, 한국/미국)
- API 응답: p95 < 500ms
- 검색 응답: < 300ms

### SEO (공개 서비스 필수)
- 원료/제품 페이지 SSG/ISR
- sitemap.xml 자동 생성
- Open Graph + structured data (JSON-LD)
- 한국어 기본, 영문 제품은 영문 페이지

### 보안
- Supabase RLS: 소비자는 `is_published = TRUE`만 접근
- Admin: Supabase Auth + 역할 기반 접근 제어
- API 키: Supabase Vault 또는 환경 변수

### 가용성
- Vercel: 자동 스케일링, 글로벌 CDN
- Supabase: 자동 백업, PITR (Pro)
- 목표 가용성: 99.5% (MVP)
