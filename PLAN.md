# 영양제·건강기능식품 비교 분석 웹페이지 — 프로젝트 계획서

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프로젝트명** | 내 영양제 비교 (My Supplement Compare) |
| **목적** | 비의료인이 영양제·건강기능식품을 쉽게 비교·선택할 수 있도록 돕고, 무분별한 복용으로 인한 부작용을 예방한다 |
| **대상 사용자** | 일반 소비자 (비의료 전문가) |
| **핵심 가치** | 정확성 · 비교 용이성 · 부작용 경고 · 접근성 |

---

## 2. 핵심 기능 목록

### 2.1 영양제 데이터베이스
- 제품별 기본 정보 (제품명, 제조사, 가격대, 제형)
- 주요 성분 및 함량
- 식약처 인정 기능성 표시 여부
- 일일 권장 섭취량 대비 비율 (%)

### 2.2 비교 기능
- **성분 기반 비교**: 같은 성분(예: 비타민D)을 포함한 제품끼리 나란히 비교
- **목적 기반 비교**: 사용 목적(예: 뼈 건강, 면역력)별로 관련 제품 그룹핑
- **가성비 비교**: 1일 섭취 비용 기준 정렬

### 2.3 부작용·주의사항 경고 시스템
- **과다 복용 경고**: 상한섭취량(UL) 초과 시 시각적 경고
- **성분 중복 경고**: 여러 제품 동시 선택 시 같은 성분 중복 합산 표시
- **상호작용 정보**: 영양제 간, 영양제-의약품 간 알려진 상호작용 표시
- **특수 집단 주의**: 임산부, 수유부, 어린이, 특정 질환자 주의사항

### 2.4 개인화 기능
- 현재 복용 중인 제품 목록 관리 ("내 영양제함")
- 선택 제품의 총 영양소 합산 대시보드
- 초과/부족 영양소 시각화

## 5. 기술 스택

| 영역 | 기술 | 선정 이유 |
|------|------|-----------|
| **프레임워크** | Next.js (React) | SSG로 빠른 로딩, SEO 최적화 |
| **스타일링** | Tailwind CSS | 빠른 UI 개발, 반응형 지원 |
| **데이터 저장** | JSON 파일 (정적) | DB 없이 Git으로 데이터 관리 가능, 초기 단계에 적합 |
| **차트/시각화** | Chart.js 또는 Recharts | 영양소 비율 시각화 |
| **배포** | Vercel | Next.js 최적 호스팅, 무료 플랜 |
| **언어** | TypeScript | 데이터 구조의 타입 안전성 |

## 6. 페이지 구조

```
/                          → 메인 (검색, 목적별 바로가기)
/ingredients               → 성분 전체 목록
/ingredients/[id]          → 성분 상세 (효능, 부작용, 권장량)
/products                  → 제품 목록 (필터·정렬)
/products/[id]             → 제품 상세
/compare                   → 비교 도구 (최대 4개 제품 나란히 비교)
/my-supplements            → 내 영양제함 (복용 목록 관리 + 합산 분석)
/guide                     → 영양제 선택 가이드
/guide/interactions        → 상호작용 조회
/guide/cautions            → 특수 집단별 주의사항
/disclaimer                → 의료 면책 조항
```

## 7. UI/UX 핵심 원칙

1. **경고는 눈에 띄게**: 과다복용·상호작용 경고는 빨간색 배너로 즉시 표시
2. **쉬운 언어**: 전문 용어 사용 시 툴팁으로 쉬운 설명 제공
3. **모바일 우선**: 대부분의 사용자가 모바일로 접근할 것을 가정
4. **의료 면책**: 모든 페이지 하단에 "이 정보는 의료 조언이 아닙니다" 명시
5. **출처 투명성**: 모든 정보에 출처(식약처, 학술논문 등) 표기

## 9. 법적·윤리적 고려사항

| 항목 | 대응 방안 |
|------|-----------|
| **의료법 저촉 방지** | "의료 조언이 아님" 면책 조항 필수 표시. 진단·처방 기능 절대 불포함 |
| **광고·홍보 배제** | 특정 제품 추천·광고 없음. 객관적 성분·함량 정보만 제공 |
| **데이터 출처** | 식약처 공공데이터, 한국영양학회 기준, 학술논문 등 신뢰 출처만 활용 |
| **개인정보** | 서버 저장 없음 (localStorage만 사용). 계정 시스템 미도입 |
| **저작권** | 제품명·제조사는 사실 정보로 기재 (상표권 침해 아님) |

## 10. 데이터 수집 전략

1. **식약처 건강기능식품 공공데이터 API** 활용

4. 부작용·상호작용 정보는 학술 문헌 기반으로 보수적으로 기재
